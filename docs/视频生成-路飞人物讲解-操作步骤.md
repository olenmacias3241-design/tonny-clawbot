# 视频生成（路飞人物讲解）详细操作步骤

本文档说明如何在本项目中生成「路飞人物讲解」类视频，以及可选的口播动嘴（SadTalker）配置。

---

## 一、前置条件（必选）

1. **Python 环境**  
   项目能正常运行（已安装依赖、能启动服务）。

2. **ffmpeg**  
   - 用于合成视频、转码音频。  
   - 检查：终端执行 `ffmpeg -version`，有输出即已安装。  
   - 未安装：  
     - macOS：`brew install ffmpeg`  
     - 其他系统请搜索「ffmpeg 安装」按系统安装。

3. **edge-tts**  
   - 用于把旁白文字转成配音。  
   - 一般随项目依赖已安装；若报错可执行：`pip install edge-tts`。

---

## 二、步骤一：放置路飞人物图（必做）

1. 准备一张**路飞**（或任意你想用的讲解人物）图片，建议竖版或方形、人物居中，格式为 **PNG 或 JPG**。

2. 将图片放到项目中的模板目录，并**固定文件名**：
   - 路径：`data/templates/`
   - 文件名二选一：**`video_avatar.png`** 或 **`video_avatar.jpg`**  
   - 完整示例：`data/templates/video_avatar.png`

3. 若之前已经给过程序一张路飞图，可直接从资源目录拷贝到模板目录，例如：
   ```bash
   cp "/path/to/你的路飞图.png" "/path/to/claw-bot-ai/data/templates/video_avatar.png"
   ```

4. 确认文件存在：
   ```bash
   ls -la data/templates/video_avatar.png
   # 或
   ls -la data/templates/video_avatar.jpg
   ```

放好后，生成视频时会自动使用该图作为人物画面（无 SadTalker 时为静态图 + 配音 + 字幕；有 SadTalker 时为口播视频）。

---

## 三、步骤二：在应用中生成视频

### 方式 A：从「生成视频」页面

1. 启动项目（如 `python main.py` 或项目提供的启动方式）。
2. 浏览器打开应用，在导航里进入 **「生成视频」** 页面（一般为 `/video`）。
3. 在页面上：
   - **选择内容来源**：勾选「使用当前对话内容」则用某次聊天的内容；或取消勾选，在下方文本框**粘贴/输入**要讲解的正文。
   - **勾选「同时生成视频文件」**（必须勾选才会生成 MP4）。
4. 点击「生成」。
5. 等待生成结束，页面会给出：
   - 视频脚本 / 分镜（可下载）
   - **下载视频** 或 **下载配音 (MP3)** 的链接（生成成功时出现视频链接，失败时可能只有配音链接）。

### 方式 B：从聊天里触发

1. 在聊天里先和机器人对话，聊出你希望做成视频的内容。
2. 发送类似指令，例如：
   - 「根据上面的内容生成视频」
   - 「把上面的内容做成视频，并生成视频文件」
3. 若机器人回复里提供「生成视频」的链接，点进去会跳到生成视频页面，并带上当前对话 ID；在该页面勾选「同时生成视频文件」后生成即可。

无论哪种方式，只要：
- 已放置 `data/templates/video_avatar.png`（或 .jpg），且  
- 勾选了「同时生成视频文件」  

生成的 MP4 就会是「路飞人物画面 + 配音 + 字幕」；若已配置 SadTalker，则会优先生成口播（人物动嘴）版本。

---

## 四、步骤三（可选）：安装 SadTalker 实现口播动嘴

若希望路飞**跟着配音动嘴**（口型同步），需要在本机安装 SadTalker。以下为详细步骤。

### 4.1 环境要求

- **Python 3.8+**（建议与项目同一环境）。
- **GPU**（推荐）：显存建议 ≥4GB，可大幅加快生成。
- **磁盘**：预留数 GB 用于仓库和模型。

### 4.2 克隆仓库并安装依赖

在**项目根目录**下执行（将 `/path/to/claw-bot-ai` 换成你的项目实际路径）：

```bash
cd /path/to/claw-bot-ai
git clone https://github.com/OpenTalker/SadTalker.git
cd SadTalker
pip install -r requirements.txt
```

若使用 conda/venv，请先激活对应环境再执行 `pip install`。

### 4.3 下载模型到 checkpoints

1. 在 SadTalker 目录下确保有 `checkpoints` 文件夹：
   ```bash
   mkdir -p checkpoints
   ```

2. 打开 SadTalker 官方安装说明，按其中要求下载模型：  
   https://github.com/OpenTalker/SadTalker#installation  

3. 将下载的模型文件解压到 `SadTalker/checkpoints/` 下，使目录结构类似：
   ```
   SadTalker/
   ├── inference.py
   ├── checkpoints/
   │   ├──  ...（官方要求的各子目录与 .pth 等文件）
   ```

具体要下载哪些文件、放在哪些子目录，以 **OpenTalker/SadTalker 官方 README 的 Installation 为准**。

### 4.4 让项目找到 SadTalker（二选一）

**方式 1：放在项目根目录（推荐）**

- 克隆时已放在项目根下，且目录名为 **`SadTalker`**（或 **`sadtalker`**），则无需再配置，程序会自动检测。
- 示例结构：
  ```
  claw-bot-ai/
  ├── SadTalker/          # 或 sadtalker/
  │   ├── inference.py
  │   └── checkpoints/
  ├── data/
  ├── src/
  └── ...
  ```

**方式 2：使用环境变量**

- 若 SadTalker 放在其他路径，可设置环境变量（启动应用前执行）：
  ```bash
  export SADTALKER_DIR=/绝对路径/SadTalker
  ```
- 再将上述命令写进 shell 配置（如 `~/.zshrc`）或启动脚本，以便每次生效。

### 4.5 验证是否被识别

- 放置好路飞图并勾选「同时生成视频文件」后，生成一次视频。
- 若日志或行为表明调用了 SadTalker（例如生成时间明显变长、或日志中有 SadTalker 相关输出），且最终得到的是口播视频，即表示配置成功。
- 若未安装或模型缺失，程序会自动退回「静态路飞图 + 配音 + 字幕」，不影响使用。

---

## 五、步骤四：下载与查看生成的视频

1. 生成成功后，在「生成视频」页面或聊天回复中会看到 **「下载视频」** 或类似链接。
2. 点击即可下载 MP4 到本机。
3. 视频文件也会落在项目的 **`data/generated/`** 目录下，文件名为 `video_xxxxxx.mp4`，可直接用本地播放器打开查看。

若生成失败，页面上可能会提示「可下载配音 MP3」：表示 TTS 已生成音频，但视频合成失败（例如 ffmpeg 报错或 SadTalker 未装/报错），可先下载 MP3 试听，再按上文检查 ffmpeg、人物图路径和 SadTalker 配置。

---

## 六、简要检查清单

| 项目           | 说明 |
|----------------|------|
| 路飞图         | 已放到 `data/templates/video_avatar.png` 或 `.jpg` |
| ffmpeg         | 已安装，终端可执行 `ffmpeg -version` |
| edge-tts       | 已安装，项目能正常跑即可 |
| 生成视频时     | 已勾选「同时生成视频文件」 |
| SadTalker（可选） | 已克隆、已装依赖、已下载模型到 `checkpoints/`，且路径为 `SadTalker` 或已设 `SADTALKER_DIR` |

按上述步骤操作后，即可得到路飞人物讲解视频；需要动嘴效果时，完成步骤四（SadTalker）即可。
